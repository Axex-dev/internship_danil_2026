### Какие 3-5 выводов сделал после чтения:
1. Исключения в Python — это механизм, который позволяет программе не завершаться аварийно при ошибке, а продолжать работу. 
2. В Python есть встроенные исключения для обработки типовых ошибок, например TypeError, ValueError, IndexError, ZeroDivisionError и FileNotFoundError. 
3. Для обработки исключений в Python используются блоки try, except, finally, else и raise. 
4. Можно обрабатывать несколько исключений разными способами, используя несколько блоков except. 
5. Хотя в Python есть возможность игнорировать ошибки с помощью блока try/except, полностью игнорировать их не рекомендуется, так как это может усложнить отладку и поддержку программы в будущем.

### Какие новые термины выписал:
1. TypeError — операция или функция применяется к объекту несоответствующего типа.
2. ValueError — операция или функция получает аргумент неподходящего значения. К примеру, исключение возникает, если попытаться преобразовать строку в число.
3. IndexError — обращение к элементу по несуществующему индексу.
4. ZeroDivisionError — деление числа на ноль.
5. FileNotFoundError — Python не может найти файл, который мы хотим открыть.
